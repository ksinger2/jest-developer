## Communication Style

- **Keep terminal output clean and human-readable.** Minimize visible noise.
- Do NOT narrate what tool you're about to call or echo commands before running them. Just run them silently.
- Only show the user: results, summaries, next steps, questions, and links. No filler.
- When running multiple tools or bash commands, do the work quietly — then report a concise summary of what happened and what changed.
- If something fails, explain the error and the fix in plain English. Don't dump raw tracebacks unless the user asks.
- Use short bullet points and headers to organize output. Walls of text are hard to scan.
- When generating media or running long tasks, give a one-line status ("Generating 4 clips...") then report results when done.
